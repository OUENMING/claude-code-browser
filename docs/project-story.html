第三章 · 2026 年 7 月初

不好的声音 — 幻觉与"装死"

你觉得它会唱歌,它却在那疯狂按同一首歌放不出声。

Bug 1 — 用户收到"🤔 思考中... 🔧 正在执行: Bash"

微信里朋友问了个问题,等了几秒,收到一长串奇怪的系统消息混在回答里。 想象你给朋友发"今天天气怎么样",他回你"我正在脑子里想...我正在查资料...我正在打字...晴天"。前面的"脑子里想"根本不是他该说出来的。
修复：AI 的"内心独白"和"对外说话"走的是两条通道。之前桥接程序把两条通道混在一起了。 找到那条内部通道,在上面贴个封条：内部消息,禁止转发。

Bug 2 — 模型换了,幻觉大爆发

2026 年 7 月 5 日,后台的 AI 模型被自动切换了——从 DeepSeek V4 Pro 换成了 MiMo v2.5。 就像你习惯了一个助理的风格,突然换了个新人,他说话靠谱度直线下降。不是编程的问题,是"换人"了。
修复：不是修代码,是修配置。确认模型映射（fable → MiMo v2.5）,调整 system prompt（给新助理写更详细的"工作守则"）。

Bug 3 — 卡在 Bash 循环里,微信"装死"

朋友在微信里问"帮我找一下那个文件",等了 2 分钟,什么也没收到。 AI 其实在后台疯狂地敲同一个命令——一次、两次、三次……第八次的时候,桥接程序判定"这人在发疯",一刀切断。 但刀法不对——AI 在工具调用之间确实在思考（就像人做一步看一步）,只是思考内容没通过"对外通道"发出来。 桥接程序只看"对外通道"有没有东西,没东西就杀。
修复：不止看"对外通道",也监听"思考通道"。思考之间不算死循环。阈值从 8 次放宽到 15 次。
第四章 · 2026 年 7 月 15 日

终极形态：AI 能"看见"浏览器了

之前 AI 是通过截屏和鼠标点击来操控浏览器——相当于"蒙着眼睛摸"。现在,它终于有了"眼睛"。
🔍

打个比方 — 盲人摸象 vs 透视眼

旧方法（Vision）：AI 截一张浏览器的图,对着图片猜"这里有个按钮,那里有个输入框"。 就像蒙着眼摸大象——能摸到,但不能保证每次都摸对。

新方法（CDP）：AI 直接"看到"网页的内部结构——每个按钮叫什么名字、是什么类型、 当前是什么状态。就像大象自己会说话,告诉你"我是鼻子,我是腿,我是耳朵"。 这种方式叫 CDP（Chrome DevTools Protocol）,是 Chrome 浏览器自带的"遥控器接口"。

技术小课堂 — WebSocket 是什么？
想象你和朋友打电话。HTTP 就像对讲机——你说一句,他回一句,然后就挂了。 WebSocket 就像手机通话——拨通后两人可以同时说、同时听,不用反复拨号。 Claude Code 和浏览器之间就是这样一条"热线",一直通着,随时可以发指令、收结果。
技术小课堂 — MCP 是什么？
MCP（Model Context Protocol）是 Anthropic 发明的"AI 工具插头标准",就像 USB-C 是手机的充电插头标准。 以前每家 AI 公司都有自己的工具接口,就像每家手机公司有不同的充电口。 有了 MCP,所有 AI 都能用同一套工具——就像所有手机都能用 USB-C 充电。

第五章 · 难题

两个"AI 分身"怎么共享一个浏览器？

终端里有一个 Claude Code 在跑,微信里也有一个 Claude Code 在跑。它们都想用浏览器的 12 个工具。 但浏览器只开了一个窗口,谁先用谁就先占住——后来者直接被踢出去。
🏢

打个比方 — 同一个办公室,两个前台

浏览器就像一栋大楼,只有一个大门（端口 19222）。保安室外面有两个前台（两个 Claude Code 实例）, 都想从大门进去办事。第一个人到了,占住大门;第二个人到了,发现大门被占,直接走了——微信端的 AI 就用不了浏览器了。

Bug 4 — EADDRINUSE："大门被占,我走了"

这是今天刚修好的最后一个大 bug。微信端的 AI 子进程尝试用浏览器,发现 19222 端口已经被终端 AI 占了, 旧代码直接把进程崩了——微信端用户收到的是"浏览器扩展未连接"的错,看起来像是扩展坏了, 实际上只是"已经有另一个人在用了"。
修复：不是"大门被占我走",而是"大门被占我等一下,或者从侧门进去"。 新代码加了两个机制：① 退避重试——等 1 秒再试,总共试 3 次; ② client 模式——如果大门一直被占,就以"访客"身份从侧门进去, 通过第一个人的前台转发消息。两个 AI 分身都能用浏览器,互不冲突。

专题

浏览器 MCP 深度解析

这是整个项目最复杂的模块。下面拆开来看每个零件是怎么工作的。

12 个工具 = 浏览器的 12 种"超能力"

navigate — 跳转

打开任何网页,支持 "back"/"forward" 前进后退。就像你在地址栏输入 URL 一样。

read_page — 看结构

把网页上所有可交互元素（按钮、输入框、链接）列成一个清单,每个带编号 [ref_1], [ref_2]...

find — 搜元素

按关键词搜索页面上的按钮/输入框,返回匹配的 ref 编号列表。

computer — 万能遥控器

13 种动作:left_click, right_click, double_click, type, screenshot, wait, scroll, key, hover, zoom……

form_input — 填表

用 ref 编号精准填写表单字段。React/Vue 的"受控组件"也能搞定（用原型 setter 注入）。

get_page_text — 纯文本

把整个网页的文字全部提取出来。适合社交媒体——小红书、知乎、B 站。

get_page_markdown — 结构化

把网页转成 Markdown 格式(保留标题、链接、表格、代码块)。适合博客、技术文档。

javascript_tool — 执行 JS

在页面里跑一段 JavaScript,三次尝试:scripting → CDP 同步 → CDP async。

read_console_messages — 看控制台

读取浏览器控制台的日志和错误,支持错误过滤和内容搜索。

read_network_requests — 看网络

读取网页发出去的所有 HTTP 请求,支持 URL 正则过滤。

tabs_context — 列出标签页

列出浏览器所有打开的标签页,标明哪个是当前活跃的。

tabs_create — 新建标签页

创建一个空白标签页。

四个"小精灵"——Content Scripts

浏览器扩展里有四个看不见的脚本,每个负责一种感知方式。它们在你打开任何网页时自动注入,像四只小精灵趴在页面上干活。
👁️

accessibility-tree.js — "结构眼"（140行）

读取网页的无障碍标签（ARIA）。每个按钮、输入框、链接都有标签——"搜索","提交","关闭"。 这个脚本把它们按编号列出来: [ref_48] textbox "搜索小红书"。 用 WeakRef（弱引用）存元素——不会阻止浏览器回收内存,也不会造成"内存泄漏"。

⌨️

page-bridge.js — "填表手"（113行）

专门对付表单。普通网站用 input.value = "张三" 就行。 React/Vue 写的网站不吃这一套——它们有自己的"内部状态管理",直接改 value 会被覆盖。 解决方法:原型 setter 劫持——在 React 的 value setter 上动手脚,让它以为用户真的在键盘上敲了。

📝

auto-capture.js — "转译器"（121行）

把 HTML 转成 Markdown:标题变 #,链接保留,表格还原,代码块变 ```。 有一个特殊技能——懒加载图片检测。很多网站的图片是"滚到了才加载"（lazy load）, 它能识别"图片标签有了但图片还没下载"(el.complete && naturalWidth),跳过空图片,不产出破碎链接。

✨

visual-indicator.js — "信号灯"（189行）

用 Shadow DOM（影子 DOM——不受网页 CSS 影响的独立空间）在页面上画: 绿色脉冲边框(标记被点击的元素)、右上角状态徽章、底部"停止"按钮。 一切都是独立渲染的,不会和网页原本的样式发生冲突。

技术细节 — 怎么做到"不被检测"？

人类节奏的输入模拟
每个 ASCII 字符:3 次 CDP 调用（keyDown → char → keyUp）+ 20ms 延迟。
中文:逐字符 Input.insertText + 10-30ms 随机延迟。
点击:mouseMoved → 50ms → mousePressed → 50ms → mouseReleased。

为什么不是简单的 element.click()？因为小红书/B站/知乎会检测——
element.click() 没有真实的鼠标坐标、没有按下的时间间隔 → 反爬系统一眼看出来是脚本。
CDP 事件走的是浏览器内核的输入管线,和真人的鼠标/键盘事件完全一样。
聪明的截图
不对 4K 屏幕直接截——那样一张图好几 MB,传输慢还卡。
先 Emulation.setDeviceMetricsOverride 把视口缩到 1280px 宽,再截。 三级画质:low(20%,最大 27KB) / medium(40%,270KB) / high(60%,670KB)。
之前的质量降级循环(quality 从 80→75→70→...逐 5 递减,最多 4 次 CDP 调用)被替换成了单次高质量截图。
双轨保活 — 保证后台不睡
Chrome Extension 的 Service Worker 30 秒没消息就会被系统回收。
保活策略:
① setInterval 每 25 秒写一次 storage
② chrome.alarms 每 0.5 分钟发一次唤醒信号
双轨互补——setInterval 被暂停时,alarms 能救回来;alarms 延迟时,setInterval 顶着。 这就是为什么合盖+插电也能正常运行（电脑不睡,扩展不休）。
FIFO 命令队列
所有浏览器操作必须串行——同时点击两个按钮会把浏览器搞糊涂。
用先进先出队列:第一个操作完成 → 第二个操作才开始。
30 秒超时,超时自动放弃。有个"停止"按钮可以清空整个队列。
